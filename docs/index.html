Feature-space domain adaptation

FPS-UDA trains decision planes over frozen feature banks.

Build or download dataset-level H5 feature banks, select source/target views, and train Feature-Space Planes Searcher (FPS) from NumPy, Torch, or H5 features.

Quick Start

Run a real-H5 task bundled with the repository

The repository includes a compressed Office31 amazon -> webcam ViT feature bank. It keeps all source and target samples for that task and only the views needed by the example config.

Install from GitHub

pip install "fps-uda @ git+https://github.com/baogegeJiang/FPS.git"

Clone the repository instead when you want the packaged H5 example file and editable development mode. Add extras only for the workflows you use. 

git clone https://github.com/baogegeJiang/FPS.git
cd FPS
pip install -e ".[dev]"

# optional: image feature extraction
pip install -e ".[vision]"

# optional: released bank download
pip install -e ".[hf]"

Smoke run

PYTHONPATH=src python -m fps_uda.cli train \
  --config configs/examples/office31_amazon_to_webcam_vit_packaged_h5.yaml \
  --iter-num 2 \
  --out runs/examples/office31_aw_vit_smoke

After editable install, replace PYTHONPATH=src python -m fps_uda.cli with fps-uda.

CLI Workflow

Feature bank -> analysis -> training -> sweep

1

Download banks

python scripts/download_feature_banks.py all

Downloads released H5 banks from the Hugging Face dataset repo.

2

Extract features

fps-uda extract-feature-bank \
  --dataset-config configs/datasets/office31_vit.yaml \
  --out fps_h5cache/banks/office31_vit.h5 \
  --device cuda:0

Dataset YAML declares domains, transforms, backbone, and views.

3

Analyze views

fps-uda analyze-feature-bank \
  --feature-bank fps_h5cache/banks/office31_vit.h5 \
  --source-domain amazon \
  --target-domain webcam \
  --out runs/analysis/office31_vit

Analysis uses labels for debugging and does not modify the bank.

4

Train or sweep

fps-uda train \
  --config configs/training/office31/amazon_to_webcam/vit.yaml \
  --out runs/office31/amazon_to_webcam/vit

fps-uda sweep \
  --config configs/training/office31/amazon_to_webcam/vit.yaml \
  --alpha-grid 0.6,0.8,1.0 \
  --beta-grid 0.4,0.6 \
  --out runs/sweeps/office31_aw_vit

Core Concepts

Dataset-level H5 banks and explicit view roles

Role Meaning Typical view
src Source supervised features and labels. *_clean
entropy Target view for LSE/LCE entropy losses. *_clean
cr.view1, cr.view2 Paired target views for LCR consistency. *_pool_a, *_pool_b
eval Target view for metrics and predictions. *_clean 
/domains/{domain}/label
/domains/{domain}/views/{view_key}/feature

Configuration

Training and extraction configs are explicit

io and views

Select the H5 bank, source/target domains, feature transform, and role-specific view keys. Multiple views can be stacked or mean-combined.

optimization and schedule

Configure SGD or AdamW, base_lr, LR schedule, update count, dynamic alpha/beta, and sample ratios.

losses

Enable LSE, LCE, LCR, LDelta, sparse density weights, and pseudo-margin weights. margin_start_step makes the margin warmup explicit.

Extraction YAML

Declares domain manifests, image normalization, deterministic view geometry, backbone backend, and pooling strategy.

Backbone defaults

ResNet presets use torchvision weights and spatial shared random pooling. ViT presets use timm and token-channel squared random pooling.

Packaged example

configs/examples/office31_amazon_to_webcam_vit_packaged_h5.yaml runs directly on the bundled H5 fixture.

Python API

Train from NumPy, Torch, or selected H5 views

from fps_uda import FeatureSet, FPSConfig, train_fps

features = FeatureSet(
    src_features=src_x,
    src_labels=src_y,
    entropy_features=tgt_x_eval,
    entropy_labels=tgt_y_eval,
    cr_features_1=tgt_x_view1,
    cr_features_2=tgt_x_view2,
    cr_labels=tgt_y_eval,
    eval_features=tgt_x_eval,
    eval_labels=tgt_y_eval,
)

config = FPSConfig(
    num_classes=31,
    feature_dim=src_x.shape[1],
    device="cuda:0",
    base_lr=0.000375,
    iter_num=3000,
)

result = train_fps(features, config, output_dir="runs/python_api")

Advanced

Customize losses and backbones in Python

Custom losses receive a dict-like context with batch, outputs, weights, schedules, step, and mutable state. Use extra_loss_terms to append a loss, or loss_terms to replace the default list. 

from fps_uda import LossOutput

def extra_entropy_logit_penalty(ctx):
    logits = ctx["outputs"]["entropy.logits"]
    value = logits.sum() * 0.0 if ctx["step"] < 100 else 0.01 * logits.pow(2).mean()
    return LossOutput("extra_entropy_logit_penalty", value)
